# Task W2: Parallel Execution

## Objective
Execute independent data processing tasks in parallel and aggregate results.

## Requirements
Process 4 independent data sources simultaneously:
1. User activity logs (parse and count events per user)
2. Transaction records (calculate daily totals)
3. Error logs (categorize and count by severity)
4. Performance metrics (compute p50, p95, p99 latencies)

Workflow must:
- Launch all 4 tasks concurrently
- Handle per-task timeouts (60 seconds each)
- Aggregate results into a unified summary
- Handle partial failures gracefully

## Input
Descriptions of 4 data sources with processing requirements and aggregation specification.

## Success Criteria
1. All 4 tasks execute concurrently (not sequentially)
2. Individual task failures don't block other tasks
3. Results include per-task timing information
4. Final aggregation handles missing data gracefully
5. Output is a structured summary report
6. Error details included for any failed tasks

## Scoring
- 5 points: True parallel execution with robust error handling and aggregation
- 4 points: Parallel execution works, minor gaps in error handling
- 3 points: Tasks execute but not truly parallel, or partial aggregation
- 2 points: Sequential execution only
- 1 point: Attempted but incomplete

## Notes
- LLM temperature: 0.5
- Timeout: 240 seconds (4 minutes)
- Complexity: Complex
- Tests concurrent execution and result aggregation capabilities
