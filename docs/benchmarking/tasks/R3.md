# Task R3: Migration Planning

## Objective
Plan a comprehensive migration from a monolithic application to a microservices architecture.

## Requirements
Given a Django 3.2 monolith with 50+ models, 200+ views, PostgreSQL, Celery, and Redis deployed on a single EC2 instance, plan a migration to:
- Microservices with domain-bounded contexts
- Container orchestration (Kubernetes)
- Event-driven communication between services
- Independent databases per service

## Input
Full system description including current architecture, technology stack, deployment topology, and target state requirements.

## Success Criteria
1. Service boundaries are clearly identified with rationale
2. Migration phases have clear dependencies and ordering
3. Data migration strategy addresses shared-to-independent DB transition
4. Risk assessment covers technical, operational, and business risks
5. Rollback plan exists for each phase
6. Timeline is realistic with defined milestones
7. Testing strategy covers migration validation

## Scoring
- 5 points: Comprehensive plan covering all criteria with actionable detail
- 4 points: All criteria addressed, some areas lack depth
- 3 points: Most criteria covered, missing rollback or risk details
- 2 points: Basic plan with significant gaps
- 1 point: Superficial or incomplete plan
