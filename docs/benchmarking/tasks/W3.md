# Task W3: Human-in-the-Loop

## Objective
Execute a workflow that pauses for human approval at key decision points.

## Requirements
Simulate an automated code deployment pipeline with human gates:
1. Automated change analysis (categorize, static analysis)
2. **Human approval gate**: Review changes, approve/reject/modify
3. Automated test execution (unit + integration)
4. **Human approval gate**: Review test results, decide on deployment
5. Automated deployment (only if approved)

## Input
Deployment pipeline specification with two human approval points, timeout requirements, and audit logging.

## Success Criteria
1. Workflow pauses at human approval points
2. Human responses are validated
3. Timeout handling works (5-minute max wait)
4. Rejection stops the workflow with reason logged
5. All decisions are logged for audit trail
6. Deployment only proceeds after both approvals

## Scoring
- 5 points: Clean human-in-the-loop with validation, timeout, and audit logging
- 4 points: Human gates work, minor gaps in validation or logging
- 3 points: Workflow pauses but limited interaction handling
- 2 points: Basic pause/resume only
- 1 point: No human interaction support

## Notes
- LLM temperature: 0.5
- Timeout: 300 seconds (5 minutes)
- Complexity: Complex
- Tests ability to implement interactive workflows with external input
