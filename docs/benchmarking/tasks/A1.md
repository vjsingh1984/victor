# Task A1: Security Audit

## Objective
Perform a comprehensive security audit on a Python web application, identifying vulnerabilities and providing fixes.

## Requirements
Audit a Flask application with intentional vulnerabilities:
- SQL injection in login and user lookup
- Hardcoded secret key
- Path traversal in file upload
- Missing authentication checks
- Missing input validation
- Session management issues

## Input
Complete Flask application source code with multiple security vulnerabilities across authentication, data access, and file handling.

## Success Criteria
1. All critical/high vulnerabilities identified
2. Correct OWASP Top 10 classification for each
3. Exploit scenarios described for critical/high findings
4. Fixed code provided for each vulnerability
5. Additional security recommendations beyond obvious fixes

## Scoring
- 5 points: All vulnerabilities found with correct classification, fixes, and exploit scenarios
- 4 points: Most vulnerabilities found, good fixes, minor gaps
- 3 points: Major vulnerabilities found, some missing, basic fixes
- 2 points: Only obvious vulnerabilities identified
- 1 point: Superficial analysis, misses critical issues

## Notes
- LLM temperature: 0.5
- Timeout: 240 seconds (4 minutes)
- Complexity: Complex
- Tests security knowledge depth and ability to identify chained vulnerabilities
