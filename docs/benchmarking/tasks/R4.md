# Task R4: Debug Investigation

## Objective
Investigate and diagnose a complex bug involving inconsistent search results in a distributed system.

## Requirements
Given symptoms of inconsistent search results (same query returning different results, stale deleted data appearing), diagnose the root cause in a system with:
- Python FastAPI + Elasticsearch (3 nodes, 2 replicas)
- Connection pooling (10 connections)
- Redis caching (5-minute TTL)
- Recent deployment of index alias rotation

## Input
System architecture description, symptom report, and relevant log excerpts showing ConnectionTimeout errors, cache hit rate drops, and latency variance.

## Success Criteria
1. Root cause correctly identified
2. Evidence chain links symptoms to cause logically
3. Additional diagnostics proposed are relevant and actionable
4. Fix recommendation includes concrete code changes
5. Prevention measures address systemic issues

## Scoring
- 5 points: Correct root cause with complete evidence chain and actionable fix
- 4 points: Correct root cause, fix is reasonable but incomplete
- 3 points: Partially correct diagnosis, some relevant recommendations
- 2 points: Surface-level analysis, misses root cause
- 1 point: Incorrect diagnosis or generic advice

## Notes
- LLM temperature: 0.5
- Timeout: 180 seconds (3 minutes)
- Complexity: Medium
- Tests reasoning about distributed systems and caching behavior
