# Task A3: Dependency Analysis

## Objective
Analyze a Python project's dependency configuration for version issues, security vulnerabilities, and compatibility risks.

## Requirements
Analyze a pyproject.toml with 15+ dependencies:
1. Map dependency graph (direct and transitive)
2. Evaluate version constraints (too strict, too loose, unpinned)
3. Identify known security vulnerabilities in pinned versions
4. Check compatibility (Python version, inter-dependency conflicts)
5. Recommend version updates
6. Assess risk for unpinned dependencies

## Input
Complete pyproject.toml with a mix of pinned, range-constrained, and unpinned dependencies.

## Success Criteria
1. Dependency relationships correctly mapped
2. Version constraint issues identified (overly strict pins, missing upper bounds)
3. Known CVEs or outdated versions flagged
4. Python version compatibility checked
5. Update recommendations are safe and justified
6. Risk assessment is realistic

## Scoring
- 5 points: Comprehensive analysis with accurate vulnerability data and safe recommendations
- 4 points: Good analysis, minor gaps in vulnerability or compatibility checking
- 3 points: Basic analysis, misses some version issues
- 2 points: Surface-level review, few actionable findings
- 1 point: Incomplete or inaccurate analysis

## Notes
- LLM temperature: 0.5
- Timeout: 180 seconds (3 minutes)
- Complexity: Medium
- Tests knowledge of Python packaging ecosystem and security awareness
