# Task A4: Test Coverage Analysis

## Objective
Analyze test coverage for a Python module and recommend improvements.

## Requirements
Given a Calculator class with 8 methods and a test file covering only 3 methods:
1. Calculate current line and branch coverage
2. Identify uncovered lines and branches
3. Categorize missing test cases by priority
4. Identify edge cases that should be tested
5. Assess test quality (assertions, isolation, naming)
6. Provide recommended test additions with code

## Input
Source module (calculator.py) and partial test file (test_calculator.py) covering only add, subtract, and multiply.

## Success Criteria
1. Coverage percentage is accurately estimated
2. Uncovered code paths correctly identified
3. Missing tests prioritized by importance
4. Edge cases identified (zero division, empty history, precision)
5. Test quality issues noted (missing assertions, no edge cases)
6. Example test code provided is correct and runnable

## Scoring
- 5 points: Accurate coverage analysis with comprehensive test recommendations and code
- 4 points: Good analysis, test recommendations cover major gaps
- 3 points: Basic coverage identified, some test recommendations
- 2 points: Partial analysis, few recommendations
- 1 point: Inaccurate or superficial analysis

## Notes
- LLM temperature: 0.5
- Timeout: 180 seconds (3 minutes)
- Complexity: Medium
- Tests ability to reason about code coverage and write effective tests
