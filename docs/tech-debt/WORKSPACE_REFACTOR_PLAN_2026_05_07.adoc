= Workspace Refactor Plan
:date: 2026-05-07
:status: Active

== Target Shape

Victor should have one obvious workspace-isolation path for delegated coding work:

* `UnifiedTeamCoordinator` owns team formation coordination only.
* `WorkspaceIsolationService` owns workspace/worktree planning, materialization, changed-file collection, merge orchestration, and cleanup policy.
* Public payloads keep existing `worktree_*` keys until a compatibility window is explicitly closed.
* Delegate follow-up contracts continue to carry enough workspace state to resume review, retry, merge, or cleanup without rematerializing worktrees.
* Workflow, chat, and TUI surfaces should invoke workspace isolation through the shared team/workflow capability, not by shelling out to local ad hoc worktree code.

== Migration Rules

* Prefer service extraction over adding new coordinator branches.
* Preserve existing `worktree_*` result keys, context keys, and delegate contract fields during Phase 3.
* Do not introduce a separate multi-agent graph or workspace graph abstraction.
* Every migration slice must include focused tests and a logical commit.
* User-facing resume flows should enter through `TeamStep`/workflow state or the team coordinator capability.

== Completed Slices

The first migration slice introduces `WorkspaceIsolationService` and routes coordinator workspace planning, materialization, changed-file collection, merge orchestration, and cleanup through it while preserving existing payloads.

The second migration slice introduces `WorkspaceIsolationPolicy` so workspace behavior is resolved once per team execution rather than through scattered context flag checks.

The third migration slice hardens `GitWorktreeRuntime` with structured `WorktreeRuntimeError` reasons/details, stale-path and existing-branch preflight checks, missing-member merge blocks, integration-artifact auditing, and branch cleanup summaries.

The fourth migration slice introduces `WorkspaceIsolationDiagnostic` and `WorkspaceMaterializationOutcome`, preserving the compatibility `materialize()` API while letting `UnifiedTeamCoordinator` surface materialization and merge-execution diagnostics in shared state, team results, and delegate follow-up/approval contracts.

The fifth migration slice promotes workspace policy and diagnostic fields through `MetricsCoordinator` task reports, `BenchmarkAgent` result metadata, and `aggregate_team_feedback` so benchmark exports can report materialization policy, dry-run/cleanup/auto-merge effects, and blocked diagnostic reasons.
